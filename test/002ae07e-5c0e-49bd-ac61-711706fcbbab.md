# ParamBinding uniqueness shape

SHACL-lite shape declaring two invariants on every `test__ParamBinding` instance:

1. **Cardinality:** exactly one `_paramName` (xsd:string) and exactly one `_paramValue`.
2. **Uniqueness:** within a single `test__CommandTestCase` that references this binding via `_inputBinding`, all `_paramName` values across sibling bindings must be unique. Catches accidental duplicate input bindings to the same `exocmd__Command_param`.

Engine: `npx @kitelev/exocortex-cli validate schema --shapes-mode` exercises this constraint. Positive fixture: two bindings with different names → valid. Negative fixture: two bindings with same name under one parent → SHACL violation reported.

## Implementation note (2026-05-23 Phase 1 Task 1.4)

The `sh__uniqueWithinParent` frontmatter key is an Exocortex-specific SHACL-lite extension (parent-relative uniqueness — analogous to SQL `UNIQUE (parent_id, paramName)` composite key). Standard SHACL has `sh:uniqueLang` and `sh:disjoint` but not "unique-within-parent-asset" out-of-the-box; the engine handles this as a custom rule. Pre-existing precedent: similar `_paramName` uniqueness shapes exist for `exocmd__Command_param` siblings within `exocmd__Command` — same engine, same key.
